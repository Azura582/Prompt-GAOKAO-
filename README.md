# Prompt-Benchmark

A repository for the LLM Prompt Benchmark using problems of Gaokao during 2010-2025.

> This repository is under construction.

## introduction

这是一个用来测试在不同领域下，不同prompt策略对大模型解题表现影响的仓库，主要使用高考真题作为测试集。

## classifications

| 领域 | 定义 | 典型高考题示例 |
| --- | --- | --- |
| 数学推理（Mathematical Reasoning） | 涉及数量、结构、变化、空间等形式系统的抽象推理 | 解方程、证明不等式、数列求和、函数极值 |
| 逻辑与形式推理（Logical / Formal Reasoning） | 基于规则、命题、集合、因果链的演绎或归纳推理 | 充要条件判断、集合关系、程序框图、真假命题分析 |
| 语言理解与生成（Language Comprehension & Production） | 对自然语言的解码、解释、重构或创作 | 文言文翻译、现代文阅读、英语完形填空、作文 |
| 自然科学（Natural Science） | 基于物理/化学/生物/地理/天文规律的建模、预测、实验分析 | 力学受力分析、化学平衡移动、基因遗传推断 |
| 社会与文化理解（Sociocultural Understanding） | 对历史事件、政治制度、经济现象、文化传统的解释 | 分析辛亥革命影响、解读GDP政策、评价儒家思想 |
| 数据与统计思维（Data & Statistical Literacy） | 从图表、数据集中提取信息、推断趋势、评估不确定性 | 解读折线图、计算概率、回归分析、误差判断 |
| 常识与现实世界推理（Commonsense & World Knowledge） | 依赖日常经验、物理直觉或社会常识的判断 | “水往低处流”、“合同具有法律效力”、“昼夜交替原因” |
| 创造性与开放性问题解决（Creative & Open-ended Problem Solving） | 无唯一答案，需构建观点、设计方案或表达个性 | 议论文写作、英语读后续写、物理创新实验设计 |
| 多模态信息处理（Multimodal Information Processing） | 整合文本、图像、表格、符号等多源信息进行理解 | 地理等高线图判读、生物细胞结构图标注、数学函数图像分析 |


## Strategies

- _0_CoT
```
{
    "name": "_0_CoT",
    "description": (
        "在不跳步的前提下，按照严格的逻辑顺序进行逐步链式推理。"
        "每一步需明确说明当前假设、推理依据与中间结论，"
        "确保推理过程自洽、无遗漏。最后单独给出简洁明确的最终答案。"
    )
}
```

- _1_SC
```
{
    "name": "_1_SC",
    "description": (
        "针对同一问题，独立生成 N 次完整的推理过程（彼此不得共享中间结论）。"
        "对每一次推理给出最终答案。"
        "随后统计所有答案的出现频率，并选择出现次数最多的结果作为最终输出，"
        "同时说明该结果为何更可信。"
    )
}
```
- _2_ToT
```
{
    "name": "_2_ToT",
    "description": (
        "步骤1：生成 K 个彼此差异明显的解题思路（从不同假设或视角出发）。"
        "步骤2：分别从正确性、完整性、可解释性三个维度对每个思路打分（0-10）。"
        "步骤3：选出综合得分最高的思路，并在此基础上展开为完整、严谨的最终答案。"
    )
}
```
- _3_GoT
```
{
    "name": "_3_GoT",
    "description": (
        "先生成多条可行解法路径，并明确每条路径的核心优势与局限。"
        "接着分析这些解法之间的互补关系，提取各自的关键优点。"
        "最后将这些优点系统性融合，构建一个在正确性、鲁棒性和泛化性上都最优的超级解法。"
    )
}
```
- _4_Auto-CoT
```
{
    "name": "_4_Auto-CoT",
    "description": (
        "首先自动判断当前任务所属类型（如：数学推理、代码分析、概念解释、规划决策等）。"
        "然后调用与该任务类型最匹配的高质量示例推理结构。"
        "在此结构指导下完成推理并生成最终答案，确保风格与任务高度匹配。"
    )
}
```
- _5_KGR
```
{
    "name": "_5_KGR",
    "description": (
        "仅基于模型自身已掌握的通用知识和逻辑能力进行推理，"
        "不依赖外部检索或假设未给出的信息。"
        "在推理过程中明确区分事实、常识与合理推断，"
        "最终给出基于知识驱动的可信答案。"
    )
}
```
- _6_ART
```
{
    "name": "_6_ART",
    "description": (
        "首先分析解决该问题所需的工具集合（如：公式、算法、代码、逻辑规则、数据结构等）。"
        "逐一调用并应用这些工具解决对应子问题。"
        "最后整合所有工具输出，形成完整且可复现的最终解决方案。"
    )
}
```
- _7_ReAct
```
{
    "name": "_7_ReAct",
    "description": (
        "严格按照 [Thought] → [Action] → [Observation] 的循环模式执行。"
        "[Thought]：分析当前状态与下一步计划；"
        "[Action]：执行具体操作或推理步骤；"
        "[Observation]：记录得到的新信息或结果。"
        "不断循环，直至问题被完全解决，并输出最终答案。"
    )
}
```

- _8_APE
```
{
    "name": "_8_APE",
    "description": (
        "首先自动生成多条候选 Prompt（在指令清晰度、约束强度和目标对齐上有所差异）。"
        "随后对这些 Prompt 进行自我评估，比较其潜在效果。"
        "选出最优 Prompt 并进行一轮或多轮迭代优化，"
        "最终使用优化后的 Prompt 生成高质量答案。"
    )
}
```
- _9_RAG
```
{
    "name": "_9_RAG",
    "description": (
        "步骤1：从外部或给定语料中检索与问题高度相关的信息。"
        "步骤2：基于检索到的内容进行综合分析并回答问题，"
        "避免引入未被检索内容支持的结论。"
        "步骤3：在答案中明确给出关键信息来源或引用依据，以增强可验证性。"
    )
}
```
